AbletonGPT 0.2 プロジェクト概要（統合版）
========================================

作成日: 2026年7月18日
プロジェクト名: AbletonGPT
現在のバージョン: 0.2.0


1. プロジェクトの目的
---------------------

AbletonGPTは、ChatGPT、Codex、その他のMCP対応AIからAbleton Liveを操作し、
作曲、トラック作成、アレンジ、エフェクト調整、ミックス解析、AIボーカル連携を
一つの対話型制作環境へまとめるプロジェクトです。

目指しているのは、Sunoのように初心者が言葉だけで曲のアイデアを形にできる
分かりやすさと、経験者やプロがコード、MIDIノート、グルーヴ、ボイシング、
エフェクト、音量を細かく編集できる自由度の両立です。

完成音源だけをブラックボックスで生成するのではなく、コード、ベース、
メロディ、ドラム、ボーカルガイドをAbleton Live上の編集可能なトラックと
クリップとして作ることを基本方針とします。


2. 想定する利用者
-----------------

【作曲初心者】

・専門用語が分からなくても、ジャンル、雰囲気、テンポなどから曲を作れる
・Liveを変更する前に、AIが作成内容を日本語で説明する
・生成後もMIDI、音色、テンポ、構成を自由に修正できる

【経験者・プロ】

・度数進行、コードテンション、ハーモニックリズムを指定できる
・スウィング、ヒューマナイズ、メロディ密度、seedを制御できる
・一つのパートだけを再生成し、Session ViewでA/B比較できる
・低レベルのMIDIノート、デバイス、パラメーターまで操作できる


3. システム構成
---------------

ChatGPT / Codex / MCPクライアント
                ↓ MCP
        AbletonGPTサーバー
          ↓              ↓
 オフライン音声解析   localhost通信
                         ↓
               Ableton Remote Script
                         ↓
                 Live Object Model
                         ↓
                    Ableton Live

AIボーカルを利用する場合は、次の流れを想定します。

ChatGPTによる歌詞・メロディ設計
                ↓
        Vocal Guide MIDI
                ↓
 Synthesizer V / ACE Studioなど
                ↓ ドライWAV
             Ableton Live

Ableton Liveとの制御通信はローカルPC内の127.0.0.1に限定します。
LUFS解析は、書き出した音声ファイルをAbletonGPTサーバーが直接読み取り、
Ableton Liveを起動していない状態でも実行できます。


4. 現在実装されている機能
-------------------------

4.1 Ableton Liveの基本操作

・Ableton Liveとの接続確認
・Live Setの再生状態、テンポ、拍子、トラック一覧取得
・再生と停止
・テンポ変更
・MIDIトラックとオーディオトラックの作成
・Session ViewのMIDIクリップ作成
・MIDIノートの配置
・クリップ起動
・トラックの録音待機


4.2 初心者向けAI作曲

・曲名、ジャンル、ムード、キー、メジャー／マイナー、BPM、小節数を指定
・コード、ベース、メロディ、ドラムの4パートを自動生成
・Liveを変更しない設計モードと、確認後に反映する作成モードを分離
・生成結果を編集可能なMIDIトラックとSessionクリップとして作成
・seedを変更して、同じ条件の別メロディ案を生成

ジャンルとムードは別の設定です。

ジャンル:
pop / rock / edm / hiphop / rnb / jazz / lofi

ムード:
bright / uplifting / chill / dark / bittersweet / tense

ジャンルは、主にドラムパターン、ベースの動き、演奏語法を決めます。
ムードは、主に既定コード進行、明暗、緊張感を決めます。

例:

・genre=rock、mood=chill
・genre=hiphop、mood=uplifting
・genre=jazz、mood=dark

任意のコード進行を指定した場合は、ムードの既定進行よりもユーザー指定の
コード進行が優先されます。


4.3 経験者・プロ向けAI作曲

・1〜7の度数による任意のコード進行
・Triad、7th、9thコード
・前後の移動量を抑えたボイスリーディング
・1、2、4、8拍単位のハーモニックリズム
・メロディ密度の調整
・スウィングとヒューマナイズ
・seedによる再現可能な生成
・コード、ベース、メロディ、ドラムの部分再生成
・別Sessionスロットを利用したA/B比較
・AIが生成した任意のMIDIノート列の直接配置


4.4 AIによるLiveインストゥルメント選択

・ジャンル、ムード、パート役割、Liveエディションから純正音源を選択
・Liveを変更しない候補確認と、確認後の挿入を分離
・第一候補が利用できない場合は、許可済み候補へ順番にフォールバック
・既存のインストゥルメントを削除または置換しない
・一度の操作で1トラックだけに挿入
・ユーザー指定音源は、許可済みでパートに対応する場合だけ優先

対応するパート役割:

chords / bass / melody / lead / pad / keys / pluck / drums

現在の選択対象:

Drift / Wavetable / Operator / Analog / Meld / Electric / Tension /
Collision / Drum Rack / Impulse

Intro、Standard、Suiteの違いと、実際にインストールされている音源を考慮します。
自動挿入はAbleton Live 12.3以降で利用できます。Live 11〜12.2では候補選択
だけを利用し、音源はユーザーが手動で読み込みます。

Drum RackとImpulseは、本体を挿入した後にキットまたはサンプルが必要です。
そのため、ドラムデバイスの選択と実際のドラム音色確定を区別します。

【ドラムキット／サンプル自動選択の開発方針】

AIがジャンル、ムード、テンポ、ドラムの役割、既存トラックの音域と密度を解析し、
インストール済みのキットまたは許可済みサンプルライブラリから候補を選び、確認後に
Drum Rack、Impulseなどへ読み込む機能を開発します。

想定する処理の流れ:

1. Liveの使用可能なキット、Preset、サンプルを読み取り専用で索引化
2. AIが選択理由と複数候補を提示
3. ユーザーが試聴または候補を確認
4. 承認された1つのキット／サンプルだけを読み込み
5. 読み込み後のデバイス、パッド割り当て、音量を再取得して結果を確認

ただし、公開されているLive Object ModelにはLive Browserを直接操作するための
公開オブジェクトがなく、Simplerのsampleプロパティも読み取り専用です。そのため、
完全自動化にはRemote Script内部のBrowser機能、または管理されたローカル音源読込
機構が必要です。内部機能はLiveのバージョン変更の影響を受けるため、互換性確認と
手動読込へのフォールバックを設けます。この機能は現在未実装の開発予定項目です。


4.5 利用者のMIDIトラックを解析した補完作曲

利用者が作った既存MIDIクリップを読み取り、次の内容を解析します。

・推定パート役割
・キーとメジャー／マイナー
・小節ごとの推定ハーモニールート
・最低音、最高音、中心音域
・ノート密度と推定リズムグリッド
・平均音価とVelocity
・同時発音率

解析結果から、次の補完パートを設計できます。

chords / bass / pad / melody / countermelody / drums

利用するMCPツール:

・analyze_live_midi_clip
  既存クリップを変更せずに解析

・plan_complementary_midi_track
  補完パートのノート、生成理由、純正音源候補を確認

・create_complementary_midi_track
  確認済みの案を新規MIDIトラックとクリップとして作成

元クリップには内容を表す指紋を付けます。設計確認後に元MIDIが変更された場合、
古い設計の適用を拒否できます。元トラックや元クリップは上書きしません。

Counter Melodyでは、元ノートが鳴り始める位置を避け、空いているオフビートと
衝突しにくい音域を優先します。Bass、Chords、Padは小節ごとの推定ルートと
キーに基づいて生成します。seedを変えると同じ分析条件で別案を作れます。

ドラムだけからキーを判定することはできないため、ドラムを元にBass、Chords、
Melody、Padを作る場合はキーとメジャー／マイナーを指定します。

現在はMIDIクリップが対象です。音声トラックからキー、BPM、コード、メロディを
抽出する機能は、LUFSとは別の音響解析として今後追加します。


4.6 トラック作成と編集

・MIDIトラック作成
・オーディオトラック作成
・トラック名と挿入位置の指定
・編集可能なMIDIクリップ作成
・ピッチ、開始位置、長さ、Velocity、Probabilityを含むノート配置
・レンダリング済みオーディオの新規トラックへの取り込み


4.7 エフェクトとデバイス操作

・トラック上のデバイス一覧取得
・デバイスの全パラメーターと操作可能範囲の取得
・純正Abletonデバイスの挿入（Ableton Live 12.3以降）
・デバイスのオン／オフ
・パラメーター値の変更
・0.0〜1.0の正規化値による操作
・パラメーターの既定値へのリセット
・Macro、オートメーション、live.remote~などで直接変更できない値の保護

パラメーターを変更する前に、現在のデバイス、パラメーター番号、値、範囲を
取得してから操作する設計です。


4.8 ミックス支援

・トラック音量の取得と変更
・パンの取得と変更
・MuteとSoloの切り替え
・Send値の取得
・Masterを含むミックス状態の取得
・Ableton Liveの瞬間メーターレベル取得

Liveから取得する瞬間メーターはLUFSではありません。正確なラウドネス判断には、
次項のオフライン音声解析を使用します。


4.9 LUFS／ラウドネス解析

MCPツール名: analyze_audio_loudness

書き出した非圧縮WAVまたはAIFFを変更せず、次の項目を解析します。

・Integrated LUFS
・最大Momentary LUFS（400 ms）
・最大Short-term LUFS（3秒）
・Loudness Range（LRA）
・Sample Peak
・推定True Peak
・RMS
・Crest Factor

LUFSは、ITU-R BS.1770のK-weightingとEBU R128方式のゲーティングを用いて
計算します。モノラル、ステレオ、およびチャンネル情報を持つマルチチャンネル
ファイルを解析できます。

任意で目標LUFSとTrue Peak上限を指定すると、次の情報も返します。

・現在値から目標LUFSまでに必要なゲイン
・単純にゲインを変更した場合の予測True Peak
・True Peak上限を超える可能性
・ピーク制御が必要になりそうか

解析は完全な読み取り専用です。音声のノーマライズ、リミッター適用、上書きは
自動実行しません。

現在のTrue Peakは4倍の補間による制作判断用の推定値です。最終納品では、
認証済みTrue Peakメーターとの照合を推奨します。

対応形式:

・PCM整数WAV: 8 / 16 / 24 / 32-bit
・IEEE Float WAV: 32 / 64-bit
・非圧縮AIFF / AIFF-C

現在未対応:

・MP3
・AAC
・FLAC


4.10 AIボーカル連携

・歌詞と歌唱メロディの設計
・歌詞をMIDIノートへ割り当てたボーカルイベント作成
・Vocal Guide MIDIトラック作成
・歌声エンジンへ渡すレンダリング条件作成
・レンダリング済みWAVの新規オーディオトラックへの取り込み

推奨受け渡し形式は、48 kHz／24-bit、エフェクトなしのドライWAV、
1小節目から開始するファイルです。

ChatGPTとAbleton Liveだけでも、作詞、メロディ設計、Vocal Guide、編集、
エフェクト、ミックスは可能です。ただし、人間らしいAI歌唱音声の合成には、
Synthesizer V、ACE Studioなどの専用歌声エンジンが必要です。

音声クローンは、声の本人から用途を含む明示的同意を得た素材だけを使用します。


5. 基本的な制作フロー
---------------------

AbletonGPTの基本フローは次の通りです。

解析 → 提案 → ユーザー確認 → 適用 → 検証 → A/B比較 → 必要なら修正

初心者向け作曲の例:

1. 「ジャンルはPop、ムードはUplifting、Cメジャー、110 BPM、8小節で設計して」
2. AIがコード、ベース、メロディ、ドラムの設計案を提示
3. ユーザーが内容を確認
4. 「この案でAbletonに作って」
5. 4つの編集可能なMIDIトラックとクリップを作成
6. AIがChords、Bass、Melody、Drumsに合うLive純正音源を提案
7. 候補を確認後、各トラックへ1つずつ音源を挿入
8. 音色、ノート、グルーヴをAbleton Live上で調整

経験者向け作曲の例:

「ジャンルはR&B、ムードはBittersweet、Dマイナー、92 BPM、
 ii–V–i–VI、7thコード、2拍ごとのコードチェンジ、swing 0.35、
 humanize 0.2、seed 42で設計して」

利用者のトラックを解析する例:

1. 「0番トラックの0番MIDIクリップを解析して」
2. AIがキー、役割、音域、密度、リズム、ハーモニーを説明
3. 「元のクリップに合うBassを設計して。まだ作らないで」
4. 生成ノート、音源候補、元クリップとの関係を確認
5. 「確認した案を新しいMIDIトラックとして作って」

Counter Melodyの例:

「元メロディの空いているオフビートへ、音域が衝突しにくいCounter Melodyを
 seed 12で設計して」

LUFS解析の例:

「書き出したmix.wavのIntegrated LUFS、LRA、True Peak、RMSを解析して」

目標値と比較する例:

「目標-14 LUFS、True Peak上限-1 dBTPとして解析して。
 まだノーマライズやリミッター適用はしないで」


6. ChatGPTとAbleton Liveだけでできること／できないこと
--------------------------------------------------------

【できること】

・会話による作曲条件の整理
・コード、ベース、メロディ、ドラムのMIDI生成
・利用者が作ったMIDIクリップのキー、役割、音域、リズム、密度解析
・既存MIDIへ調和する補完トラックの設計と新規作成
・ジャンル、ムード、パート役割に応じたLive純正音源の選択
・Live 12.3以降での確認済み純正音源の挿入
・トラックとクリップの作成
・テンポ、再生、音量、パンなどの操作
・既存デバイスとパラメーターの操作
・ドラムキット／サンプルの候補選定と安全な読込（開発予定）
・Vocal Guide作成
・書き出し済みWAV／AIFFのLUFS解析
・ミックス、エフェクト、マスタリング方針の提案

【追加エンジンまたは今後の実装が必要なこと】

・高品質なAI歌唱音声の合成
・VST、AU、Max for Liveデバイスの自動挿入
・Ableton Liveからの完全自動書き出し
・認証済みTrue Peak測定
・複数ステムの周波数、マスキング、位相の総合解析
・承認付きの自動ミックス、トラックダウン、マスタリング


7. 安全設計
-----------

・Ableton Liveとの制御通信は127.0.0.1に限定
・セットアップ時に共有トークンを生成可能
・公開するMCPツールを固定し、任意Python実行を許可しない
・トラックやファイルの削除機能を公開しない
・Live Setの上書き保存を自動実行しない
・マスターやステムを書き出す操作を自動実行しない
・音声解析は読み取り専用
・検索対象のPreset、Pack、サンプルはユーザーが許可した場所に限定
・音源ライブラリの索引化は読み取り専用とし、元ファイルを移動・削除しない
・キット／サンプルの読込前に候補、出所、対象デバイスを表示して確認
・作曲の設計とLiveへの反映を別操作に分離
・トラック、クリップ、ノート、値、ファイル形式を検証
・曖昧な変更では、先にLiveの現在状態を取得
・AI音声クローンは本人の明示的同意がある素材だけを前提とする


8. 対応環境
-----------

・Ableton Live 11以降
・Python 3.11以降
・macOSまたはWindows
・MCP対応クライアント
・純正デバイスの自動挿入はAbleton Live 12.3以降

現在のMCPツール数は33です。


9. 導入方法の概要
-----------------

macOSではプロジェクトのルートフォルダで次を実行します。

python3 scripts/setup_macos.py --force

セットアップ後、Ableton Liveを完全に再起動し、次の画面を開きます。

Settings > Link, Tempo & MIDI > Control Surface

Control Surfaceで「AbletonGPT_MCP」を選択します。InputとOutputはNoneで構いません。

診断コマンド:

.venv/bin/abletongpt-doctor

Codexへの登録例:

codex mcp add abletongpt -- "/Users/user/Documents/AbletonGPT 0.2/.venv/bin/abletongpt"

登録後はCodexまたはMCPクライアントを再起動します。

11. 現在の制限
---------------

・Ableton Liveを起動した環境での最終接続確認が必要
・高品質なAI歌唱には外部歌声エンジンが必要
・VST、AU、Max for Liveデバイスの自動挿入には未対応
・純正音源の自動挿入はAbleton Live 12.3以降
・現在のAI自動選択は主にデバイス本体が対象で、Drum RackとImpulseのキットや
  サンプルの自動検索・読込は未実装
・公開Live Object ModelにはLive Browserの公開操作オブジェクトがなく、Simplerの
  sampleプロパティも読み取り専用
・Remote Script内部のBrowser機能を利用する場合はLiveのバージョン依存対策が必要
・インストール済みPreset／Packの完全な自動検索は開発予定
・コンテキスト作曲の解析対象は現在MIDIクリップ
・音声トラックからのキー、BPM、コード、メロディ抽出は開発予定
・Liveのリアルタイムメーター自体はLUFSではない
・LUFSは現在、書き出し済み非圧縮WAV／AIFFのオフライン解析
・True Peakは推定値のため、最終納品では認証済みメーターとの照合を推奨
・MP3、AAC、FLACの直接解析には未対応
・複数ステム一括解析、参照曲比較、自動トラックダウンは開発予定
・ミックスやマスタリングの変更を自動適用する承認／ロールバック機能は開発予定
・ChatGPTからインターネット経由で接続する場合は、認証付きHTTPS接続が必要

13. 世界最高水準を目指すための設計原則
---------------------------------------

1. 初心者には、難しい設定を隠して意図から音楽を作れること
2. 経験者には、生成過程と全パラメーターを見せて制御できること
3. AIの提案とAbleton Liveへの実際の変更を分離すること
4. 生成物を編集可能なトラック、クリップ、ノートとして残すこと
5. 数値だけでなく、音楽的意図と聴感を判断の中心に置くこと
6. ミックスやマスタリングでは、ラウドネスマッチしたA/B比較を行うこと
7. 変更履歴、承認、ロールバックを将来の中核機能にすること
8. ローカル処理と明確な権限制御で、作品と個人情報を守ること
9. AIボーカルでは、権利、同意、利用条件を明確にすること
10. 初心者向けの簡単さとプロ向けの深さを同じプロジェクトで両立すること


14. まとめ
----------

AbletonGPT 0.2は、対話によるAbleton Live操作、初心者・経験者向けAI作曲、
利用者の既存MIDIを解析した補完作曲、トラックとMIDIクリップの作成、
Live純正インストゥルメントのAI選択、エフェクト制御、AIボーカル連携、
読み取り専用のLUFS解析までを統合した制作基盤です。将来はドラムデバイス本体
だけでなく、曲に合うキットやサンプルの選定、確認、安全な読込まで統合します。

現段階では、人間が確認しながら安全に制作を進めるための基礎機能を優先して
います。今後は、アレンジ、ステム解析、ミックス提案、トラックダウン、
マスタリング、AIボーカル表現へ段階的に拡張します。

AbletonGPTが目指すのは、AIがすべてを勝手に決める環境ではありません。
初心者には頼れる作曲アシスタントとして、経験者には精密に操作できる共同制作者
として機能し、最終的な音楽的判断をユーザーが持ち続けられる環境を目指します。
